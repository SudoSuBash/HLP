2. 浏览器驱动相关
2.1 为什么需要 Chrome 浏览器驱动（chromedriver）
在使用 Selenium 自动化操作 Chrome 浏览器时，chromedriver 充当了 Python 代码与 Chrome 浏览器之间的桥梁。Python 代码通过 chromedriver 向 Chrome 浏览器发送指令，如打开网页、点击按钮、输入文本等，从而实现自动化操作。因此，要让打包后的程序在其他 Windows 系统上正常运行，必须确保该系统上安装了合适的 chromedriver。
2.2 确认 Chrome 浏览器版本
在下载 chromedriver 之前，需要先确认目标 Windows 系统上安装的 Chrome 浏览器版本。你可以按照以下步骤进行操作：
打开 Chrome 浏览器。
点击浏览器右上角的三个点（菜单按钮）。
在弹出的菜单中，选择 “帮助” -> “关于 Google Chrome”。
浏览器会自动检查并显示当前安装的 Chrome 版本号，例如 119.0.6045.160。
2.3 下载合适的 chromedriver 版本
根据确认的 Chrome 浏览器版本，从 ChromeDriver 官方下载页面 下载对应的 chromedriver 版本。具体步骤如下：
打开链接https://sites.google.com/chromium.org/driver/，你会看到一个版本列表。找到与目标系统 Chrome 浏览器版本最接近的 chromedriver 版本。通常，主版本号相同即可兼容，例如 Chrome 版本为 119.x.x.x，则下载 119.x.x.x 版本的 chromedriver。
在对应版本的下载列表中，选择适合 Windows 系统的 chromedriver_win32.zip 文件进行下载。
下载完成后，解压缩 chromedriver_win32.zip 文件，你会得到一个名为 chromedriver.exe 的文件。
2.4 验证 chromedriver 是否配置正确
将修改后的 .exe 文件和 chromedriver.exe 文件准备好后，在目标 Windows 系统上运行 .exe 文件。如果程序能够正常打开 Chrome 浏览器并访问指定的网页，说明 chromedriver 配置正确；如果出现错误，可能是 chromedriver 版本不兼容或路径配置错误，需要重新检查并调整。
通过以上步骤，你就可以确保在其他 Windows 系统上正确配置 Chrome 浏览器驱动，使打包后的程序能够正常运行。
